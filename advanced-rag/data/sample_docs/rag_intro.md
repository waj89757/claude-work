# RAG 技术介绍

## 什么是 RAG？

RAG（Retrieval-Augmented Generation，检索增强生成）是一种将信息检索与文本生成相结合的技术。它通过在生成回答之前先检索相关文档，让大语言模型（LLM）能够利用外部知识来生成更准确、更可靠的答案。

## RAG 的工作流程

1. **用户提问**：用户输入一个问题或查询
2. **检索**：系统从知识库中检索与问题相关的文档片段
3. **增强**：将检索到的文档作为上下文添加到提示词中
4. **生成**：LLM 基于问题和检索到的上下文生成答案

## RAG 的优势

### 减少幻觉
大语言模型有时会生成看似合理但实际上错误的信息（称为"幻觉"）。通过提供真实的参考资料，RAG 可以显著降低幻觉的发生率。

### 知识可更新
传统的 LLM 知识是在训练时固定的，更新知识需要重新训练。而 RAG 系统只需要更新知识库，无需修改模型，大大降低了知识更新的成本。

### 可追溯性
RAG 可以提供答案的来源引用，让用户能够验证信息的准确性，增强可信度。

### 领域适应性
通过更换不同的知识库，同一个 RAG 系统可以快速适应不同的专业领域。

## RAG 的核心组件

### 1. 文档处理
- 文档加载：支持 PDF、Word、HTML、Markdown 等多种格式
- 文本分块：将长文档切分成适合检索的小片段
- 元数据提取：保留文档的标题、来源、时间等信息

### 2. 向量化
- 使用 Embedding 模型将文本转换为向量表示
- 常用模型：OpenAI text-embedding、BGE、M3E 等
- 向量维度通常在 384-4096 之间

### 3. 向量存储
- 向量数据库：Chroma、Milvus、Pinecone、Weaviate 等
- 支持高效的相似度搜索
- 可以存储和检索百万级甚至亿级向量

### 4. 检索策略
- 稠密检索：基于向量相似度
- 稀疏检索：基于关键词匹配（如 BM25）
- 混合检索：结合两者的优势

### 5. 生成模型
- 使用 LLM 基于检索到的上下文生成答案
- 需要设计合适的提示词模板
- 控制生成的格式和风格
